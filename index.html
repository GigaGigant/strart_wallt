Пилот

Апрель — июль 2026. PeakPoints + кросс-обмен с 3 внешними ПЛ на двух курортах Кавказа.

Архитектура пилота

Касса бренда → ПЛ-платформа (PremiumBonus / своя ПЛ бренда) → Wallty (клиринг / обмен)

Wallty

Суперапп для туриста + бэкенд клиринга. Кошелёк, обмен баллов, аналитика, взаиморасчёты между ПЛ

PeakPoints

ПЛ Кавказ.РФ. Бренды без своей ПЛ подключаются сюда. Якорь экосистемы на Кавказе

PremiumBonus

Движок «под капотом» PeakPoints: процессинг бонусов, интеграция с кассами (iiko, R-Keeper, 1С, Эвотор — 35+)

Два сценария работы с брендами

Сценарий 1: У бренда нет ПЛ

  • Подключаем к PeakPoints через PremiumBonus
  • Касса бренда интегрирована с PremiumBonus (35+ интеграций с кассами)
  • Турист называет телефон → касса начисляет/списывает бонусы PP

Сценарий 2: У бренда есть своя ПЛ

  • Бренд остаётся на своей ПЛ
  • Через Wallty происходит кросс-обмен бонусов
  • Турист: Wallty → «Вступить» в ПЛ бренда → перевёл PP в бонусы бренда → на кассе списал
  • Клиринг и взаиморасчёт автоматически на стороне Wallty

Продукт (MVP)

Для туриста (суперапп Wallty)

  • Вкладка «Горнолыжка Кавказа» с PeakPoints как якорем
  • Карта курорта с заведениями-партнёрами, фильтры по категориям
  • Кошелёк — все карты ПЛ и баланс в одном месте
  • Обмен баллов — конвертация PP в бонусы других ПЛ
  • QR-код для идентификации на кассе
  • История — где был, какие бонусы получил/потратил
  • Геймификация: ачивки, маршруты-квесты, streak
  • Регистрация — телефон + SMS, 30 сек, автоматом аккаунт PeakPoints

Для бренда (два ЛК)

  • ЛК PeakPoints (от PremiumBonus): транзакции, начисления, статистика ПЛ, настройка кэшбека
  • ЛК Wallty: клиринг, взаиморасчёты, аналитика клиентов через платформу

Для Кавказ.РФ (дашборд Wallty)

  • Турпоток в реальном времени
  • Тепловая карта распределения туристов (ключевая метрика!)
  • Средний чек, частота покупок, популярные категории
  • Сравнение Эльбрус vs Мамисон
  • Динамика по дням/неделям

Экономика пилота (модель СберСпасибо)

  • Бренд начисляет клиенту бонусы → оплачивает их Wallty
  • Клиент списывает бонусы у другого бренда → Wallty компенсирует
  • Раз в месяц — взаиморасчёт (клиринг). Wallty рассчитывается с каждым брендом напрямую
  • Breakage (20-30% неиспользованных бонусов) = доход платформы
  • На пилоте: условно бесплатно для брендов (с отображением цифр). Комиссия 3-5% за кросс-конвертацию между ПЛ

Продуктовые изменения

Балловая система

  • 1 PP = 1 руб по ценности
  • Юридически — бонусная скидка (не электронные деньги, не требует лицензии ЦБ)
  • Отражается в чеке как скидка, код 1354

Механика начисления / списания

  • Бренд сам выбирает: кэшбэк баллами ИЛИ скидка на месте
  • Кросс-бренд: баллы начисленные в одном бренде можно тратить в другом

Клиринг (модель СберСпасибо)

  • Партнёр платит за начисленные баллы
  • PeakPoints компенсирует списания
  • Раз в месяц — взаимозачёт по актам
  • Breakage (20-30% нереализованных баллов) = доход платформы

Сгорание баллов

  • Базовые пользователи: 12 месяцев
  • Подписчики (золотая карта): 16 месяцев

Два режима работы с брендами

  • Интегрированный: кассы (iiko, r-keeper) → автоматическая передача суммы чека
  • Ручной: бренд вводит сумму → сканирует QR → оплата % раз в месяц по отчёту

Онлайн-партнёры (конец фазы 2)

  • Такси, маркетплейсы, страховые и др.

Январь 2027 — декабрь 2027. Масштабирование на всю Россию.

Модель входа в регион

  1. Находим якорного партнёра (администрация / крупный бренд с влиянием)

  5. Wallty даёт технологию + операционную экспертизу + кейс PeakPoints
  6. Прибыль: 50+1 / 49

Объединение в холдинг + гиганты

2028. Корпоративная консолидация и интеграция с крупнейшими игроками.

1. Корпоративное объединение

  • Создание АО «Wallty Холдинг» (конвертация из модели А в модель Б)
  • Региональные ООО → дочки холдинга (50+1% у холдинга)
  • Якоря остаются миноритариями 49% в своих дочках
  • Единая финансовая отчётность

2. Единый кошелёк

  • Общий баланс — все баллы всех программ в одном экране
  • Кросс-региональный обмен: накопил на Кавказе → потратил в Сочи (курс 1:1)
  • Кросс-региональный клиринг через холдинг
  • Единая золотая карта на все регионы

3. Глубокая интеграция с гигантами

  • Обмен баллов: Яндекс Плюс ↔ Wallty, СберСпасибо ↔ Wallty
  • Совместные акции
  • Платёжная интеграция (СБП, банковские приложения)
  • Маркетплейсы (Ozon, WB)

Примечание: локальные гиганты подключаются в дочки уже на фазах 2-3. Федеральные — по мере роста базы. Фаза 4 — это интеграция обмена с их программами лояльности.

4. Подготовка к международной экспансии

  • Аудит финансов
  • Юридическая структура для международки
  • Мультиязычность, мультивалютность в технологии

План Б (если пилот не взлетит)

Б1: Пивот в B2B SaaS

  • Та же технология как «конструктор лояльности» для отелей/ресторанов
  • Каждый бренд получает свою программу, не коалиция
  • Монетизация: SaaS подписка 5-15К/мес

Б2: Пивот в аналитику

  • Продажа данных о турпотоке региональным администрациям, туроператорам, девелоперам
  • Кавказ.РФ уже партнёр

Б3: Агентская модель

  • Агрегатор скидок/купонов для курортов (а-ля Groupon для туристов)
  • Комиссия с использованного купона
  • Не нужна интеграция — бренды дают промокоды
Уроки

Ключевые уроки (из провала Wallty v1)

1
Никогда не overengineer MVP за 3 недели, не за 12 месяцев
2
Якорный партнёр решает cold start Без Кавказ.РФ повторится та же история
3
Ценность ДО интеграции Бренд должен получить пользу с первого дня, без сложной интеграции

Технология — следствие продукта, не наоборот Сначала доказать что работает, потом масштабировать стек
5
Данные важнее фич Аналитика турпотока = ценность для Кавказ.РФ с первого дня
